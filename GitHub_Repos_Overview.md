## Discord Channel Creation Guide

### Step 1: Create Channel in Discord

**For each repository:**

1. **Go to Discord** and navigate to your server
2. **Server Settings** ‚Üí Channels ‚Üí Create Channel
3. **Fill in details:**
   - Channel Name: Use `#channel-name` format
   - Channel Type: Text or Voice
   - Category: Choose from list
   - Description: Brief purpose of channel
   - Overwrite Policy: If name taken, choose new name
4. **Click Create Channel**

### Step 2: Configure Channel Settings

**After channel creation:**

1. **Channel Settings** ‚Üí Overview
   - Set channel topic and description
   - Set slow mode (if needed)
   - Set age restriction (if needed)
   - Set region (if applicable)

2. **Roles** ‚Üí Members ‚Üí Roles
   - Add "Contributor" role with write access (for your team)
   - Add "Moderator" role (if you have community members)
   - Add "Bot" role with write access (for automated bots)

3. **Webhooks** ‚Üí Integrations ‚Üí Discord Webhooks
   - Enable "Server Members Intent"
   - Create webhook (GitHub ‚Üí Discord)
   - Configure "Server Members Content"
   - Configure "Server Members Emojis"
   - Set up message formatting

4. **Moderation** ‚Üí AutoMod (if applicable)
   - Enable "Explicit Content Filter" (if public channel)
   - Set blocked keywords
   - Enable spam filters
   - Configure slow mode (if needed)

### Step 3: Send Initial Announcement

**For each channel:**

1. **Pin Channel Message:**
   ```markdown
   #channel-name
   
   üìã #channel-name Channel Created
   
   [Detailed purpose description]
   
   [Key resources, links, or information]
   
   Feel free to ask questions, share learnings, and collaborate on [repository purpose]!
   ```

2. **Set Welcome Message:** Enable "Send Welcome Message" in channel settings to welcome new members

### Step 4: Set Up Webhooks (Optional)

**For repositories with GitHub integration:**

1. **Create Webhook:** Discord ‚Üí Integrations ‚Üí New Webhook
2. **Select Repository:** Choose your GitHub repository
3. **Select Events:**
   - Push (new commits and branches)
   - Pull Request (new and merged)
   - Release (tags published)
   - Deployment (status updates)
4. **Configure Content Type:** "Server Members Intent" or similar
5. **Test Webhook:** Send test payload or wait for actual event
